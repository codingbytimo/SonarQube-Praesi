Warum Codequalität prüfen?

Never change a running system oder sowas

Warum Codequalität prüfen?

  • Team-Standards und Best Practices fördern
  • Früherkennung von Fehlern & Sicherheitsrisiken
  • Verbesserung von Lesbarkeit und Wartbarkeit
  • Höhere Gesamtqualität erzielen
  • Unterstützt Teamstandards & Clean Code
  • Technische Schulden vermeiden

Was sind technische Schulden?

"Ich mach das mal schnell..."

  • Metapher für bewusst eingebaute oder ignorierte Schwächen im Code, um schneller zu sein
  • Kurzfristige Produktivität kann langfristige Kosten verursachen

Was ist SonarQube?

  • (Open Source) Tool zur statischen Codeanalyse
  • Findet Bugs, Sicherheitslücken & Code Smells
  • Visualisiert Metriken in einem Webinterface
  • Kann in CI/CD-Umgebungen eingebunden werden
  • SonarQube for IDE (früher SonarLint)

Was ist Code Smell?

  • Kein direkter Bug – aber möglicher "Designfehler"
  • Oft schwierig zu warten oder zu testen
  • Beispiele:
    • Lange Methoden
    • Unnötig duplizierter Code
    • Unnötige Komplexität
    • Unbenutzte Variablen

Wie funktioniert SonarQube?

  • Führt clientseitige, statische Codeanalyse durch
  • Ergebnisse werden an lokalen oder Remote-Server gesendet
  • Analyse anhand durch sog. Quality Profiles
  • Visualisierung von Ergebnissen und Metriken in Dashboards

Statische Codeanalyse?

  • Quellcode wird analysiert, ohne dass er ausgeführt wird
  • Code wird geparsed (Syntaxbaum)
  • Anschließende Analyse anhand von Regeln

Quality Profiles in SonarQube

  • Definieren, welche Regeln bei der Analyse gelten
  • Pro Programmiersprache kann ein Profil aktiv sein
  • Vordefinierte Profile vorhanden (z. B. "Sonar way")
  • Eigene Profile möglich (anpassbar je Projekt/Team)
  • Aktive Regeln beeinflussen direkt, was als Fehler gemeldet wird

Quality Gates & Metriken

  • Quality Gate: Definiert, ob Code „akzeptabel“ ist
  • Typische Metriken:
    • Test Coverage
    • Duplikate im Code
    • Bugs
    • Sicherheitsrisiken
  • Custom Gates möglich
    (z. B. „Coverage < 80%=Fail“)

SonarQube Editions

Community Edition

  • Kostenlos & Open Source
  • Basisfunktionen für statische Codeanalyse
  • Unterstützt ein paar Sprachen (Java, JavaScript, Python, ...)
  • Findet Bugs, Code Smells, Sicherheitslücken
  • Ideal für kleine Teams oder Lernzwecke

Developer Edition

  • Erweiterte Spracheunterstützung (C, C++, Swift, Kotlin, ...)
  • Branch-Analyse (z. B. Feature Branches separat analysieren)
  • Pull Request Decoration für GitHub, GitLab, Bitbucket
  • Sicherheitsanalysen auf Basis von OWASP Top 10

Enterprise Edition

  • Projekt-Portfolios zur Verwaltung vieler Projekte
  • Zentrale Dashboards & Reporting für Teams und Abteilungen
  • Benutzer-, Rechte- und Rollenverwaltung
  • Skalierbar für große Organisationen

Data Center Edition

  • Hochverfügbarkeit (High Availability)
  • Cluster-Setup mit Load Balancing
  • Für große Unternehmen mit kritischer CI/CD-Infrastruktur
  • Monitoring & Ausfallsicherheit im Fokus

Demo mit Java Projekt

Kleines Maven-Projekt mit schlechtem Code und lokaler SonarQube Instanz

1. SonarQube installieren & starten

  • Download unter: sonarsource.com
  • ZIP entpacken
  • Start über Terminal: 
    ./bin/[os]/sonar.sh start
  • Im Browser öffnen: http://localhost:9000

2. Java-Projekt vorbereiten

  • Maven-Projekt mit pom.xml verwenden
  • Optional: Java-Version setzen mit Compiler-Plugin
  • Beispiel: 
    <source>11</source>

pom.xml Setup

Innerhalb vom build-tag einfügen


		  
			org.sonarsource.scanner.maven
			sonar-maven-plugin
			3.9.1.2184

3. Projekt in SonarQube anlegen

  • Auf "Create Project" klicken
  • Project Key und Name festlegen
  • Token generieren (wird bei der Analyse gebraucht)

4. Analyse mit Maven starten

Im Projektordner folgenden Befehl ausführen:


				mvn clean verify sonar:sonar \
				  -Dsonar.projectKey=DEIN_KEY \
				  -Dsonar.host.url=http://localhost:9000 \
				  -Dsonar.token=DEIN_TOKEN

→ Startet die Analyse & sendet Ergebnisse an SonarQube

5. Analyse im Browser ansehen

  • Ergebnisse unter http://localhost:9000 einsehbar
  • Qualitäts-Gate prüfen für eigenen Usecase

Fazit

Fazit

  • SonarQube ist ein gutes Werkzeug zur Verbesserung von Codequalität
  • Gut in Projekte bspw. über Maven integrierbar
  • Hilft Teams, Standards einzuhalten und Fehler zu bemerken

Allerdings...

  • Statische Codeanalyse kann nicht alle Fehler erkennen
  • Manche Fehler treten erst bei Laufzeit auf
  • Die Analyse ist nur so gut wie eure festgelegten Regeln

Danke für eure Aufmerksamkeit!
